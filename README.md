# 서울 본사 월간 모니터링 리포트 (5월, 개편판)

단일 HTML을 HTML / CSS / JS / CSV로 분리한 버전. **그래프 1개 = CSV 1개** 원칙.

## 폴더 구성
```
bonsa/
├─ index.html      마크업
├─ style.css       디자인 (한 속성당 한 줄)
├─ app.js          CSV 읽어 차트/표 그리는 코드
├─ start.bat       윈도우 서버 자동 실행 (포트 8000, 더블클릭)
└─ data/
   ├─ temp_zone.csv       3-1 구역별 일평균 온도 (+실외최고기온)
   ├─ temp_ctrl_low.csv   3-2.1 1·2·3층·별관 제어기 온도 (+실외)
   ├─ temp_ctrl_6f.csv    3-2.2 6층 제어기 온도 (+실외)
   ├─ temp_ctrl_7f.csv    3-2.3 7층 제어기 온도 (+실외)
   ├─ oper_zone.csv       4-1 층별 일평균 가동시간
   ├─ oper_bar.csv        4-2 층별 근무내/외 총가동시간
   ├─ increase_work.csv   4-3 전월대비 증가(근무시간)  열: 구역,전월,당월
   ├─ increase_off.csv    4-4 전월대비 증가(근무외)    열: 구역,전월,당월
   └─ holidays.csv        주말·공휴일 날짜 목록 (x축 라벨 빨강)
```

## 실행
폴더에서 `python -m http.server 8000` → http://localhost:8000
또는 start.bat 더블클릭 (포트 8000).

## CSV 수정 가이드
- **온도 CSV 4종 (temp_*)**: 첫 열 날짜(1~31), 나머지 열은 구역/제어기 이름.
  맨 끝 "실외최고기온" 열은 검은 점선으로 모든 온도 그래프에 자동 오버레이됨.
  결측은 빈칸. 범례는 헤더에서 자동 생성됨.
- **oper_zone.csv**: 첫 열 날짜, 나머지 층별 일평균 가동시간(h).
- **oper_bar.csv**: 한 줄=한 구분. 열은 구분,근무내,근무외 (총가동시간 h).
- **increase_work.csv / increase_off.csv**: 한 줄=한 구역. 열은 구역,전월,당월.
  증감(당월-전월)과 증감률은 코드가 자동 계산하고 증감 큰 순으로 정렬함.
- **holidays.csv**: 주말·공휴일 날짜를 한 줄에 하나씩. 여기 적힌 날짜는
  모든 일자별 그래프의 x축에서 빨간색으로 표시됨. **매달 이 파일만 고치면 됨.**

## 주의
- 인코딩 UTF-8 고정. start.bat은 UTF-8(BOM).
- 현재 데이터는 5월 기준 임의 더미값. 실제 그래프별 CSV가 오면 같은 헤더 형식으로 갈아끼우면 됨.
- 수기 작성 항목(index.html 직접 수정): 1번 KPI 숫자/주의 항목, 각 차트 밑 코멘트,
  2번 기기 현황 숫자, 헤더/푸터 문구.
- 실제 CSV 헤더가 더미와 다르면 app.js 매핑(컬럼명)을 맞춰야 함.
